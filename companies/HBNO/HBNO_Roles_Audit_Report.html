üî¥ Critical Security Vulnerabilities

1. NETSTOCK Consultant Role - CRITICAL THREAT

Script ID: customrole_netstock_consultant

Risk Level: CRITICAL

Excessive Permissions:

  • ‚úó FULL Manage Roles - Can create/modify any role including administrators
  • ‚úó FULL Manage Users - Can create new user accounts
  • ‚úó FULL Enable Features - Can enable/disable NetSuite features
  • ‚úó FULL Allow JS/HTML Upload - Can inject malicious code
  • ‚úó FULL Web Services Setup - Complete API control
  • ‚úó FULL OAuth Management - Can generate OAuth tokens
  • ‚úó FULL Audit Login - Can view login history
  • ‚úó No Two-Factor Authentication

Attack Scenarios:

  1. Privilege Escalation: Modify own role to add administrator permissions
  2. Account Takeover: Create new administrator accounts
  3. Code Injection: Upload malicious JavaScript to exfiltrate data
  4. API Abuse: Generate OAuth tokens for unauthorized API access

Immediate Actions Required:

  • üö® Audit all users assigned to this role
  • üö® Review all customizations created in last 90 days
  • üö® Check all OAuth tokens generated
  • üö® Review for any new roles or administrator accounts created
  • üö® Remove Manage Roles, Manage Users, and Enable Features permissions
  • üö® Enable mandatory 2FA

Phase 2: High Priority (7-30 Days) ‚ö†Ô∏è

SOD Violation Remediation

  1. Split Accountant Role (customrole_hbno_mfg_em_acct)
    • Remove journal entry creation OR approval authority
    • Remove vendor payment approval if retaining bill creation
    • Implement maker-checker workflow
  2. Split Controller Role (customrole_hbno_mfg_em_controller)
    • Remove Audit Login permission (admin only)
    • Implement workflow requiring CFO approval for period closing
    • Remove journal entry creation OR approval authority
    • Require 2FA authentication
  3. Restrict Purchasing Manager (customrole_hbno_mfg_em_purchasing_mgr)
    • Separate vendor bill creation from approval authority
    • Implement three-way match validation
    • Require approval for vendor master data changes
  4. Implement Inventory Adjustment Approvals
    • Roles: Warehouse Manager, Supply Chain Manager
    • Require Controller approval for adjustments >$500
    • Implement cycle count verification process

Remove Administrative Permissions from Operational Roles

  1. Production Roles (customrole1353, customrole1354, customrole1455)
    • Remove: Allow JS/HTML Upload
    • Remove: Custom Scripts FULL access
    • Change to VIEW only for scripts
  2. Warehouse/WMS Roles (customrole_hbno_wms_material_handler, customrole_hbno_wms_warehouse_manager_na)
    • Remove: Manage Roles permission
    • Remove: Custom Script permissions
    • Remove: Script Scheduling
    • Disable: JS/HTML Upload
    • Remove: Account Setup access
  3. Developer Roles (customrole_hbno_senior_developer, customrole_netsuite_dev_team)
    • Remove all financial transaction permissions
    • Remove OAuth management (or limit to own tokens)
    • Change web services to VIEW
    • Require script deployment approval workflow

Phase 3: Medium Priority (30-90 Days) üìã

  1. Implement IP Restrictions
    • All integration roles (NETSTOCK, Amazon, JP Morgan)
    • All third-party vendor roles (NetSuite PS)

    • NETSTOCK Integration: Set iswebserviceonlyrole = T

  3. Implement Time-Limited Access
    • NetSuite Development Team: Deactivate when project ends

  4. Consolidate Duplicate Roles

  5. Add Approval Workflows
